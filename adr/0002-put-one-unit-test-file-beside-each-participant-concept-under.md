# ADR-0002: Put one unit-test file beside each participant concept under `test/participants`

**Status:** Accepted
**Context:** The goal says “each participant”; agents could disagree whether that means only the documented core/observer participants or every class in `src/participants/`.
**Decision:** Cover every TypeScript participant class in `packages/baro-orchestrator/src/participants/`, excluding non-participant helpers `story-executor.ts` and `forwarders/index.ts`. Use exact paths `packages/baro-orchestrator/test/participants/<source-basename>.test.ts` for top-level participants and `packages/baro-orchestrator/test/participants/forwarders/<source-basename>.test.ts` for forwarders. The required source basenames are: `auditor`, `cartographer`, `claude-cli-participant`, `codex-cli-participant`, `codex-story-agent`, `conductor`, `critic`, `critic-codex`, `critic-openai`, `critic-opencode`, `critic-pi`, `finalizer`, `librarian`, `memory-librarian`, `openai-story-agent`, `opencode-cli-participant`, `opencode-story-agent`, `operator`, `pi-cli-participant`, `pi-story-agent`, `sentry`, `story-agent`, `story-factory`, `surgeon`, `surgeon-codex`, `surgeon-openai`, `surgeon-opencode`, `surgeon-pi`, plus forwarders `agent-stream`, `coordination`, `finalization`, `progress`, `story-lifecycle`, and `token-usage`.
**Consequences:** Existing tests such as `test/story-executor.test.ts` and `test/surgeon-prompt.test.ts` may remain, but they do not satisfy the new “each participant” naming/coverage requirement unless a matching file exists under `test/participants/`.
