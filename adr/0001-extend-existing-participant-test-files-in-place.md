# ADR-0001: Extend existing participant test files in place

**Status:** Accepted
**Context:** The goal is to extend test cases for all participants, and every current participant already has a matching test file under `packages/baro-orchestrator/test/participants/`. Creating a second parallel test tree or renaming files would fragment coverage and conflict with the existing ADR convention.
**Decision:** Add new coverage in the existing matching test files only. For top-level participants, use `packages/baro-orchestrator/test/participants/<source-basename>.test.ts`. For forwarders, use `packages/baro-orchestrator/test/participants/forwarders/<source-basename>.test.ts`. The required participant basenames are `auditor`, `cartographer`, `claude-cli-participant`, `codex-cli-participant`, `codex-story-agent`, `conductor`, `critic`, `critic-codex`, `critic-openai`, `critic-opencode`, `critic-pi`, `finalizer`, `librarian`, `memory-librarian`, `openai-story-agent`, `opencode-cli-participant`, `opencode-story-agent`, `operator`, `pi-cli-participant`, `pi-story-agent`, `sentry`, `story-agent`, `story-factory`, `surgeon`, `surgeon-codex`, `surgeon-openai`, `surgeon-opencode`, `surgeon-pi`, plus forwarders `agent-stream`, `coordination`, `finalization`, `progress`, `story-lifecycle`, and `token-usage`. Do not add participant tests for `packages/baro-orchestrator/src/participants/story-executor.ts` or `packages/baro-orchestrator/src/participants/forwarders/index.ts` for this goal.
**Consequences:** Implementation agents have one canonical file per participant to modify. The work should not touch Rust crates, app packages, production participant code, or package layout unless a test exposes a real bug that must be fixed.
