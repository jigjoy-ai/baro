# ADR-0005: Add behavior coverage, not constructor smoke or private-field snapshots

**Status:** Accepted
**Context:** Existing participant tests already cover one primary public contract per participant. The requested extension should improve regression coverage without freezing implementation details or requiring exhaustive branch coverage across all internals.
**Decision:** Each participant test file touched for this goal must add at least one meaningful new behavior test or materially strengthen an existing behavior assertion. The new coverage must exercise public bus, lifecycle, command, or IO contracts: observers react to the SemanticEvent or Mozaik item they subscribe to; push-style participants translate public commands; CLI wrappers prove lifecycle/result/error handling with fake process output; story-agent wrappers prove terminal `StoryResult` or outcome behavior with deterministic fake backends; forwarders prove exact emitted `BaroEvent` shapes. Do not add tests that only instantiate a class, snapshot whole objects, or assert private fields. Public accessors such as `getPhase()`, `getTouches()`, `getKnowledge()`, and `getCurrent*()` may be asserted.
**Consequences:** Agents should prefer explicit field assertions and semantic-event factory guards over broad snapshots. The extension should increase confidence in participant behavior while leaving implementation freedom intact.
