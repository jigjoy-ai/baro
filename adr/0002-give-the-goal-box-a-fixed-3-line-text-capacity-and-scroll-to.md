# ADR-0002: Give the goal box a fixed 3-line text capacity and scroll to keep the latest line visible

**Status:** Accepted
**Context:** With wrapping enabled, a goal longer than the inner width needs vertical room. The box is currently `Constraint::Length(5)` (3 inner text rows). Options: (a) grow the box dynamically with content — but the Welcome layout is a fixed stack of `Constraint::Length(...)` rows (logo, settings, help, version) and growing one row would push the others and complicate centering; (b) keep a fixed height and scroll the `Paragraph` so the tail (where the cursor is) stays visible. Lazy choice: keep the fixed layout, scroll within it.
**Decision:** Keep the goal layout row at `Constraint::Length(5)` (3 inner rows) in `welcome.rs`. Compute wrapped line count for the current `goal_input` against the inner text width (`input_area.width.saturating_sub(2)` for borders, accounting for the 1-char leading space) and, when it exceeds the 3 visible inner rows, pass `.scroll((overflow_rows, 0))` to the goal `Paragraph` so the last 3 wrapped rows (containing the cursor) are shown. When it fits, use `.scroll((0, 0))`. Do not change the surrounding `chunks` constraints or the settings/logo/help rows.
**Consequences:** The box shows up to 3 lines of the goal and scrolls as the user types past that; earlier lines scroll out of view but remain in `goal_input` and are submitted in full. No layout reflow of other Welcome elements. The wrapped-line-count computation is local to `render()`; keep it as a small inline helper in `welcome.rs`, not a shared util.
